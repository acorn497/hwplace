# 내부 상태 코드 문서
> X는 알 수 없는 숫자를 의미합니다.
> _는 해당 위치에 삽입될 수 있는 숫자를 의미합니다. 보통 의미와 함께 서술됩니다.

# API RULES
- 항상 모든 응답에 ISC가 포함되어 있습니다.
- 각 자리는 16진수입니다. (0~F, Ex-0F1C)

# 0XXX: 기본 상태 코드
## 01XX: 성공함
- 0100: 요청이 문제 없이 완료됨 (기본 성공)
- 0101: 부분 성공 (일부만 성공, 예: bulk 처리 중 일부 실패)
- 0102: 이미 처리된 요청 (중복 요청 등)

## 0_XX [1~10]: 실패함 (시스템 측 오류)
- 0200: 알 수 없는 서버 오류
- 0201: 내부 타임아웃
- 0202: 데이터베이스 연결 실패
- 0210: 외부 API 호출 실패
- 0211: 외부 API 타임아웃
- 0300: 인증/권한 부족 (토큰 만료, 권한 없음 등)
- 0301: 로그인 실패 (01XX와 구분하기 위해 시스템 측으로 분리 가능)

# 1XXX: 사용자의 입력 / 상호작용 오류
## 11XX: 잘못된 사용자 입력
- 1100: 필수 입력값 누락
- 1101: 형식 오류 (이메일, 전화번호, 날짜 등) 오류
- 1102: 길이 제한 초과/미달
- 1103: 허용되지 않는 문자 포함
- 1104: 중복값 (이미 존재하는 아이디/이메일 등)
- 1105: 값의 범위 오류 (나이 -1, 가격 음수 등)

- 11_X (구역 지정, 0~9)
  - 111X: 회원가입 폼 오류
  - 112X: 로그인 폼 오류

- 11X_ (같은 구역 내 순서)
  예시)
  - 1110: 회원가입 - 아이디 누락 또는 형식 오류
  - 111: 회원가입 - 비밀번호 누락 또는 형식 오류
  - 1112: 회원가입 - 비밀번호 확인 불일치
  - 1113: 회원가입 - 이메일 누락 또는 중복

## 12XX: 사용자 상호작용 오류
- 1200: 세션 만료
- 1201: 이미 처리된 요청 (예: 이미 투표함, 이미 결제함)
- 1202: 본인이 작성한 글이 아님 (수정/삭제 권한 없음)
- 1203: 차단된 사용자
- 1204: 일일/시간당 제한 초과 (도배 방지 등)

# 9XXX: 유지보수/특수 상태
- 9000: 점검 중
- 9001: 서비스 준비 중 (오픈 전)
- 9999: 알 수 없는 예외 (최후의 보루)
